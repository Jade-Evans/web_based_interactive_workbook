Tables

  • Tables are useful for displaying and representing relationships in tabular data (data in rows and columns).
  • Tables should not be used for layouts, for the following reasons:
    1. Table markup is more complex than layout markup; when the tags are read out by screen readers this causes confusion and limits accessibility. The added complexity also means more difficulty with writing, debugging and maintaining code than required with dedicated layout techniques.
    2. Proper layout containers such as header or div default to 100% of parent, making them automatically responsive. However, the width of tables defaults to their content, so it takes more effort to make tables define page layouts correctly.
